Flow Designer

In this project, I am working on a visual scripting tool for admins and developers.
This project is under NDA. For more information please contact me directly.

Project Overview

This project is focused on creating a GUI based tool for building contact center based processes like routing pathways that the customer experiences. The current scripting tool we have was lacking in a lot of aspects. Since this is one of the most critical functionalities of any contact center and with the current tool

Solution

We identified key gaps in the existing experience by speaking with existing users of the tool. We evaluated existing solutions out there in the market to set a competitive benchmark and defined a high-level experience before we started off with detailed designs for individual features.

Project Info

Timeline: Feb 2020 - current
Theme: Web Application, Usability Testing
Contribution: I am working with one other Interaction Designer on this project, supported by a Visual designer and User Researcher. I am working on defining certain feature sets of this product.

A TLDR version of the project

We re-designed the existing scripting tool to a significantly advanced one keeping in mind the vision for enabling a programmatic contact center.

We did a round of usability testing on the MVO features we had built for GA

We worked on enhancements based on the feedback received and continue to design and build for features beyond MVO

Process Overview

Although the work cannot be displayed on a public facing domain yet, I wanted to highlight the process followed. At a high level, this is what the it looked like:

Competitor Analysis
We visited 5 sites and talked to 17 agents, 7 supervisors, 5 administrators and 5 report designers spanning 2 weeks and across mutiple cities - Phoenix, Atlanta, Dallas, Pittsburgh and Asheville. The objective was to understand current painpoints with the solution and also observe them in the contact center settings. My primary focus was on the Agent persona.
Conceptualization and Roadmap
We narrowed down our ideas from the brainstorming session and broke them down into a phased approach. This was required to plan out the roadmap of the product. This exercise was enriching and enlightening from a UX standpoint as I worked closely with the PM to help come up with the overall breakdown of the planned features across multiple phases. We used an impact feasibility matrix to help with this breakdown apart from the data we had already gathered form our field reserach.
Layout Iterations and Information Architecture
The layout and IA was one of the most critical sections of this effort. We came up with a lot of iterations and options to arrive at an optimal solution. This work helped the UI architect plan out the development work much in advance.
Wireframing and Component level approach
We started with creating wireframes and focusing on individual components within Agent Desktop. This component based approach also helped the Development team with their work as they were planning to use Web Components as part of the overhaul in the UI Architecture.
Detailed Designs and Implementation Support
Based on agreement reached on the high-level wire-frames and the roadmap defined for the product, we started working on the detailed designs. This has been a highly iterative process with feedback received and worked upon over multiple rounds starting from a product leadership level down to Engineering level support durring implementation.

Key Learnings from this project

This remains one of the most complicated projects i have ever worked on. Since this falls heavily in the realm of programmatic experiences, I needed to learn about programmatic constructs like HTTP Request Methods. This also involved designing from a platform mentality as the flow Designer tool would be used to build upon by other internal teams and hence we had to design keeping that in mind.
