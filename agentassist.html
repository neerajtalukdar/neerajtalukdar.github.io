Agent Assist

In this project, I worked on the integration of Google's Contact Center APIs in our agent desktop application to assist agents with real-time suggestions during conversations.

Project Info

Project Members: Ryan Bloom (PM), Tony Deng (Architect)

Time: 2 months

Theme: Web Application, AI based service

Problem Statement

Agents in Contact Centers deal with lots of customer queries of various kinds. Often times, they have to reference articles, documents and FAQs to resolve these issues. A lot of time is spent on actually searching for the relevant answer. Since most contact centers are technically "Cost Centers", it becomes critical to save time, and help agents in this process of answer retrieval.

Solution

Agent Assist is an AI based integration that Google provides to Contact Center Solutions. Agent Assist provides suggestions to agents in real-time based on the context of the conversation. This assistance is primarily provided in the form of suggestions of resources which the agent could reference with ease thereby reducing the handle time.

Key Highlights

The project was done initially as a POC to deliver the MVO and ensure that Cisco got an opportunity to demo the feature during Google's Next Conference.

It was later implemented as a feature as part of our on-premise agent experience, while also being planned as a cloud offering.

Initial concpetualization was done with the idea of thinking about the possibilities of extending the current capabilities of the integration

This was my first exploration into industry based AI design work.

What propelled this effort in the first place?

Google, recently announced a suite of features in the realm of Contact Center and one of the main features was Agent Assist. Live agents are empowered with continuous support during their calls, as Agent Assist transcribes calls in real time, identifies customer intent, provides real-time, step by step assistance (recommended articles, workflows, etc.), and automates call dispositions.

The Guiding Design Principles

We set out by defining a set of guiding principles for the design. This was done in collaboration with the Product Manager and Architect. We looked at some existing literature out there on User Experience principles when it comes to building AI based services. The most noteworthy one was the The People + AI Guidebook developed by one of the teams in Google. Here is the set of guidelines we decided to follow in the design explorations.

Transparency

We should have a way of surfacing transparency to the agent about some of the inner-workings of the system.

Feedback Mechanism

There should be a way to gather feedback from agents on the suggestions being provided.

Non-intrusive

It is critical to provide suggestions in a non-intrusive manner to avoid unnecessary distractions.

Agent in Control

Agents should feel in complete control of the system at all times. Objective is to augment agents, not replace.

Seamless Error Handling

Errors are common to AI systems, there should be a graceful way to tackle any kind of issue or error.

Set Expectations

It would be important to level set the expectations to avoid disappointment in the service.
